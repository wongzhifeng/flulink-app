# FluLink道家开发法则

## 核心理念
基于《德道经》哲学指导的软件开发方法论，实现"不出户知天下"的云端开发境界。

---

## 一、核心开发法则

### 1. "治大国若烹小鲜" - 系统稳定性原则
**含义**: 治理大国如同烹饪小鱼，不能频繁翻动

**开发实践**:
- 绝不频繁修改Zion数据模型（每日≤3次结构调整）
- 保持API接口稳定性，避免破坏性变更
- 数据库Schema变更需要充分测试和回滚方案
- 服务端规则引擎保持确定性，避免随机性

**具体约束**:
```yaml
数据模型变更:
  - 每日最大变更次数: 3次
  - 变更前必须: 备份现有数据
  - 变更后必须: 验证所有API接口

API接口稳定性:
  - 版本控制: 语义化版本号
  - 向后兼容: 至少保持2个版本
  - 废弃通知: 提前30天通知
```

### 2. "天下大事必作于细" - 渐进式开发原则
**含义**: 天下的大事必须从细微处做起

**开发实践**:
- 每次Zeabur部署只变更1个文件
- 小步快跑，降低部署风险
- 功能开发采用最小可行产品(MVP)方式
- 每个功能模块独立开发和测试

**具体约束**:
```yaml
部署策略:
  - 单次部署文件数: ≤1个
  - 部署频率: 每日最多3次
  - 回滚准备: 每次部署前备份

功能开发:
  - MVP优先: 先实现核心功能
  - 迭代优化: 基于用户反馈改进
  - 模块独立: 各功能模块解耦
```

### 3. "善闭无关楗而不可开" - 生态利用原则
**含义**: 善于关闭的，不用门闩也打不开

**开发实践**:
- 用微信原生权限系统替代自定义鉴权
- 利用Zion现有数据模型，避免重复设计
- 使用Zeabur自动HTTPS，无需手动配置SSL
- 依赖微信小程序生态，减少开发工作量

**具体约束**:
```yaml
权限管理:
  - 使用: 微信原生登录
  - 避免: 自定义用户系统
  - 依赖: 微信OpenID体系

数据存储:
  - 使用: Zion免费版功能
  - 避免: 自建数据库
  - 依赖: Zion GraphQL API

部署运维:
  - 使用: Zeabur自动HTTPS
  - 避免: 手动SSL配置
  - 依赖: Zeabur监控系统
```

### 4. "不出户知天下" - 云端开发原则
**含义**: 不出门户就能了解天下事

**开发实践**:
- 完全放弃本地测试，所有验证通过Zeabur日志分析完成
- 云端开发环境，无需本地配置
- 通过日志分析了解系统运行状态
- 远程监控和调试，实现云端开发

**具体约束**:
```yaml
开发环境:
  - 禁用: 本地开发环境
  - 使用: Zeabur云端开发
  - 验证: 通过日志分析

监控调试:
  - 日志分析: Zeabur控制台
  - 性能监控: 云端指标
  - 错误追踪: 远程日志
```

```javascript
// 微信小程序限制
const constraints = {
  // 功能限制
  features: {
    animation: false,        // 禁用所有动画
```

```yaml
# Zeabur配置约束
services:
  flulink-core:
    # 资源限制
    resources:
      cpu: "0.5"           # 最大0.5核心

    # 环境约束
```

## 六、道家开发哲学总结

### 核心思想
**"让内容像流感一样自然而然地传播，让社交回归本质，让连接更有意义"**

### 实现路径
- **无为而治**: 服务端规则引擎，确定性算法
- **小国寡民**: 客户端极简设计，省流模式  
- **上善若水**: 部署流程自动化，云端开发
- **知止不殆**: 监控方案精简，关键指标

### 最终目标
实现*"不出户知天下"*的云端开发境界，4小时内完成从代码编写到生产部署的全流程。
