## 🌟 **FluLink星尘共鸣版核心设计**

### **产品愿景**
FluLink是一个遵循宇宙自然法则的数字生态系统。我们不构建社交网络，我们点亮一片星空。每个用户都是一颗拥有独特光芒的星球，通过分享与互动，释放出名为"星种"的信号。这些信号穿越数字宇宙，与其他星球产生"共鸣"，在七天的轨道周期内，汇聚成璀璨的"星团"。

### **核心世界观**
- **底层规则**：遵循《道德经》吕尚版哲学思想，万物源于"朴"，通过"阴阳"互动而"生长"
- **顶层表达**：通过"星空图谱"视觉隐喻呈现，将哲学概念转译为宇宙学术语
- **核心概念**：
  - 气(亲和力) → 共鸣：用户间潜在连接强度
  - 脉(49人组) → 星团：高共鸣度星球组成的临时星系
  - 道行(内容影响力) → 光度：星种的能量和影响力
  - 毒株(内容单元) → 星种：用户创造的可传播内容单元

## 🎯 **核心功能模块**

### **星团生成系统**
- **触发条件**: 用户A点亮用户B的星种时触发共鸣计算
- **算法流程**: 
  1. 计算A和B的共鸣值（必须≥50）
  2. 筛选47个高共鸣用户（与A、B平均共鸣值最高）
  3. 确保至少3-5个间接互动用户
  4. 按活跃度平衡（高30% + 中40% + 低30%）
  5. 确保标签多样性（同标签用户≤40%）
- **生命周期**: 7天自动解散，成员回归独立星球

### **星空图谱可视化**
- **个人视角**: 用户为中心的主星球，周围显示星团成员
- **星种辐射**: 光度≤30缓慢波纹，30-70脉冲发光，≥70星光拖尾
- **流星提示**: 高光度星种跃迁时显示来源、抵达时间、光芒预览
- **交互设计**: 点击星球查看星座档案，点击星种查看详情

### **星种演化机制**
- **光度计算**: 初始10 + 互动增量 - 时间衰减
- **互动权重**: 评论+5，转发+3，点赞+1
- **光谱演化**: 根据互动用户标签调整权重，支持负向反馈
- **跃迁条件**: 光度≥70 + 光谱匹配度≥0.6

4. **频繁提交**: 经常性commit，确保问题可回退
5. **智能提交**: 让CC写commit/pr message，简介又详细
6. **上下文管理**: 使用compact而非clear，保持上下文连续性
7. **订阅模式**: 日常使用直接订阅，避免API token计费
8. **质量门控**: 实现验证工作流，自动质量检查、静态分析、测试套件
