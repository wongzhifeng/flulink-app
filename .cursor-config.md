# FluLink MVP - Cursor 开发配置

## 🎯 项目快速启动

### 1. 环境检查
```bash
# 检查Node.js版本 (需要18.x)
node --version

# 检查npm版本
npm --version

# 检查Git状态
git status
```

### 2. 安装依赖
```bash
npm install
```

### 3. 启动开发服务器
```bash
npm run dev
# 访问 http://localhost:3000
```

### 4. 测试账号
- **用户名**: admin
- **密码**: 12345

## 🔧 开发工具配置

### Cursor 推荐设置
```json
{
  "editor.formatOnSave": true,
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  },
  "typescript.preferences.importModuleSpecifier": "relative",
  "emmet.includeLanguages": {
    "typescript": "html",

}
```

```
├── .github/                # GitHub配置
```

## 🔍 关键文件说明

### 核心配置文件
- `package.json` - 项目依赖和脚本
- `next.config.ts` - Next.js配置
- `tailwind.config.js` - Tailwind CSS配置
- `tsconfig.json` - TypeScript配置
- `Dockerfile` - Zeabur部署配置

### 重要组件
- `src/context/FluLinkContext.tsx` - 全局状态管理
- `src/app/dashboard/page.tsx` - 主仪表板
- `src/components/` - 所有功能组件

### 文档文件
- `DEVELOPMENT_TASKS.md` - 开发任务列表
- `CONTRIBUTING.md` - 贡献指南
- `ZEABUR_DEPLOYMENT.md` - 部署指南
- `memory/memory.md` - 项目记忆库

## ⚠️ 重要注意事项

### 开发规范
1. **端口**: 必须使用3000端口
2. **分支**: 功能开发使用feature分支
3. **提交**: 使用约定式提交格式
4. **审查**: 所有代码必须经过审查
5. **测试**: 新功能必须有测试

### 世界规则检查
每个功能开发前必须通过世界规则检查：
- 回归本质：是否专注社交核心功能？
- 遵循自然规律：实现是否符合自然规律？
- 追求简约与平衡：设计是否简约平衡？
- 敢作敢当：是否敢于创新和承担责任？

### 部署要求
- Zeabur平台部署
- Node.js 18 + 端口3000
- Docker配置完整
- 环境变量安全

## 🆘 遇到问题？

### 常见问题
1. **端口被占用**: 确保端口3000可用
2. **依赖安装失败**: 删除node_modules重新安装
3. **构建失败**: 检查TypeScript和ESLint错误
4. **部署失败**: 检查Dockerfile和Zeabur配置
